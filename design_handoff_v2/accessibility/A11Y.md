# Accessibilité & réponses questions ouvertes

## Mode sombre
**Statut** : preview fourni, non prioritaire v2.
- Tokens `[data-theme="dark"]` définis dans `tokens/tokens.css` (surfaces, text, soft variants).
- Tout le proto utilise des `var(--…)` donc la migration ne touche pas au markup.
- Reste à définir côté dev : graphiques Recharts (forcer `background: var(--surface)` sur tooltip/legend), PDF factures preview (inversion).
- Trigger : `<html data-theme="dark">` ou class `.dark` (Tailwind `darkMode: ['class', '[data-theme="dark"]']`).

## Accessibilité

### Contraste (WCAG AA)
| Couple | Ratio | Verdict |
|---|---|---|
| `text` #0F172A / `bg` #FAFAFA | 16.5 : 1 | AAA |
| `text-muted` #64748B / white | 5.36 : 1 | AA large seulement |
| `accent` #6366F1 / white | 4.59 : 1 | AA |
| `accent` / `accent-soft` #EEF2FF | 6.51 : 1 | AAA |
| `success` #10B981 / white | 2.48 : 1 | ⚠ pas AA texte — OK pour pill avec soft bg |
| `danger` #EF4444 / white | 3.76 : 1 | ⚠ AA large seulement |

**Actions côté dev** :
- Remplacer `text-muted` par `#475569` (ratio 7.64:1) si AAA requis.
- Ne jamais mettre de petit texte blanc directement sur `success` / `warning` / `danger` — utiliser les variants `-soft` avec texte dans la couleur pleine.

### Focus management
- Tout contrôle interactif : `outline: none; box-shadow: 0 0 0 3px var(--accent-soft);` via `:focus-visible`.
- Modals : trap focus + retour au trigger à la fermeture.
- Drawers : idem + ESC ferme.

### Navigation clavier (à implémenter côté dev)
- Sidebar : `tab` séquentiel par item, `enter` active, `arrow-up/down` navigation.
- Tables : `arrow` navigation cellule, `enter` ouvre drawer, `space` sélectionne row.
- Modals : `tab/shift+tab` cyclique, `esc` ferme.

### ARIA
- `role="navigation"` sur sidebar, `aria-current="page"` sur item actif.
- `role="dialog" aria-modal="true" aria-labelledby`… sur modals.
- `role="status"` sur toasts, `aria-live="polite"` sur banners d'info et `assertive` sur erreurs.
- Toggles : `role="switch" aria-checked`.
- Tabs : pattern tabs ARIA complet (`role="tablist" > role="tab" aria-selected > role="tabpanel"`).

### Texte alternatif
- Icônes décoratives : `aria-hidden="true"`.
- Icônes porteuses de sens : `aria-label`.
- Avatars : `alt={name}` ou `aria-label` si initiales.

### Respect prefers-reduced-motion
Voir `animations/MOTION.md`.

## Mode compact
**Non livré** — la densité actuelle est déjà élevée (table rows 44–48px, body 13px). Un mode "encore plus dense" risquerait de casser la hiérarchie typographique et les zones de clic.
Si le besoin revient : prévoir une variable `--density: normal|compact` qui module `--sp-*` et `font-size` via une classe root, pas un fork de composants.

## Breakpoints responsive
```
sm  640px   — basculement mobile → tablet
md  768px   — tablet → sidebar apparaît en drawer
lg  1024px  — sidebar permanent
xl  1280px  — 2 colonnes split
2xl 1536px  — grid 4 colonnes KPI max width
```
Mobile (< 768px) bascule vers shell dédié (`mobile.jsx`), pas du responsive CSS classique.
